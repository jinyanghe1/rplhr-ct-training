## 📦 训练阶段数据流（随机裁剪）

```
案例: CT00000000.nii.gz

Step 1: 读取原始数据
═══════════════════════════════════════════════════════════════════

5mm 输入 (低分辨率)                     1mm 标签 (高分辨率)
┌─────────────────┐                    ┌─────────────────┐
│  Shape:         │                    │  Shape:         │
│  (60, 512, 512) │                    │  (300, 512, 512)│
│                 │                    │                 │
│  60 层 ×        │                    │  300 层 ×       │
│  512 × 512      │                    │  512 × 512      │
└─────────────────┘                    └─────────────────┘
       │                                      │
       │  Z 方向比例 1:5                      │
       │  X/Y 方向比例 1:1                    │
       ▼                                      ▼


Step 2: 随机裁剪
═══════════════════════════════════════════════════════════════════

训练时每次迭代随机选择裁剪位置：

5mm 裁剪区域                              1mm 对应区域
┌──────────────────────┐                ┌──────────────────────┐
│  Z: [10:14]          │                │  Z: [53:69]          │
│  Y: [100:356]        │    ────────▶   │  Y: [100:356]        │
│  X: [200:456]        │   映射关系     │  X: [200:456]        │
│                      │                │                      │
│  4 层                │   Z×5+3        │  16 层               │
│  256×256             │                │  256×256             │
└──────────────────────┘                └──────────────────────┘

裁剪后输入尺寸: (4, 256, 256)            裁剪后标签尺寸: (16, 256, 256)


Step 3: 送入网络
═══════════════════════════════════════════════════════════════════

┌─────────────────┐        ┌─────────────────┐        ┌─────────────────┐
│  输入:          │        │                 │        │  输出:          │
│  (1, 1, 4,      │        │   TVSRN         │        │  (1, 1, 16,     │
│       256, 256) │───────▶│   Transformer   │───────▶│       256, 256) │
│                 │        │   网络          │        │                 │
│  Batch ×        │        │                 │        │  Batch ×        │
│  Channel ×      │        │   上采样 5×     │        │  Channel ×      │
│  D × H × W      │        │   (Z方向)       │        │  5D × H × W     │
└─────────────────┘        └─────────────────┘        └─────────────────┘


Step 4: 计算损失
═══════════════════════════════════════════════════════════════════

预测输出                                真实标签
┌─────────────────┐                    ┌─────────────────┐
│  Shape:         │                    │  Shape:         │
│  (1, 1, 16,     │                    │  (1, 1, 16,     │
│       256, 256) │        L1 Loss     │       256, 256) │
└─────────────────┘◀──────|y_pred ───▶└─────────────────┘
                               y_true|
                               ─────┘
                         Loss = mean(|y_pred - y_true|)
```

## 🔍 验证阶段数据流（滑动窗口切块）

```
案例: CT0000100.nii.gz (验证集)

Step 1: 读取完整卷
═══════════════════════════════════════════════════════════════════

5mm 输入体积                              1mm GT 体积
┌────────────────────────────┐          ┌────────────────────────────┐
│  Shape: (80, 512, 512)     │          │  Shape: (400, 512, 512)    │
│                            │          │                            │
│  ┌─────────────────────┐   │          │  ┌─────────────────────┐   │
│  │                     │   │          │  │                     │   │
│  │      80 层          │   │          │  │     400 层          │   │
│  │   512 × 512         │   │          │  │   512 × 512         │   │
│  │                     │   │          │  │                     │   │
│  └─────────────────────┘   │          │  └─────────────────────┘   │
└────────────────────────────┘          └────────────────────────────┘


Step 2: 滑动窗口切块
═══════════════════════════════════════════════════════════════════

由于整卷太大，切成多个小块处理：

切块策略 (vc_z=4, vc_y=256, vc_x=256, overlap_z=2):

        Z 方向 (80层)
        0    2    4    6   ...   76   78
        │────│────│────│         │────│
        ▼    ▼    ▼    ▼         ▼    ▼
       ┌────┐
       │块0 │
       ├────┤
       │块1 │
       ├────┤
       │... │
       ├────┤
       │块N │
       └────┘

实际切块 (以 Z 为例):
- 块 0:  Z[0:4]    (层 0-3)
- 块 1:  Z[2:6]    (层 2-5)  ← 重叠 2 层
- 块 2:  Z[4:8]    (层 4-7)
- ...
- 块 N: Z[76:80]  (层 76-79)

总块数计算:
- Z 方向: 80/2 = 40 个位置
- Y 方向: 512/256 = 2 个位置
- X 方向: 512/256 = 2 个位置
- 总计: 40 × 2 × 2 = 160 个小块


Step 3: 逐块推理
═══════════════════════════════════════════════════════════════════

对于每个小块:

┌─────────────────┐        ┌─────────────────┐        ┌─────────────────┐
│  输入块:        │        │                 │        │  预测块:        │
│  (4, 256, 256)  │───────▶│   TVSRN 网络    │───────▶│  (16, 256, 256) │
└─────────────────┘        └─────────────────┘        └─────────────────┘
         │                                                    │
         │  位置: (z=0, y=0, x=0)                             │
         │                                                    │
         ▼                                                    ▼
      记录位置                                           记录预测块

重复 160 次，得到 160 个预测块


Step 4: 拼接完整卷
═══════════════════════════════════════════════════════════════════

将预测块按位置拼接回完整体积：

预测体积 (拼接前)                        预测体积 (拼接后)
┌──────────────────┐                    ┌────────────────────────────┐
│  块 0  │  块 1  │                    │                            │
│  (16×  │  (16×  │      拼接          │   Shape: (400, 512, 512)   │
│  256×  │  256×  │  ───────────▶      │                            │
│  256)  │  256)  │                    │   ┌─────────────────────┐  │
├────────┼────────┤                    │   │                     │  │
│  块 2  │  块 3  │                    │   │    400 层           │  │
│  ...   │  ...   │                    │   │  512 × 512          │  │
│        │        │                    │   │                     │  │
└────────┴────────┘                    │   └─────────────────────┘  │
                                       └────────────────────────────┘

重叠区域处理: 取平均


Step 5: 计算评估指标
═══════════════════════════════════════════════════════════════════

预测体积                              GT 体积
┌────────────────────────────┐       ┌────────────────────────────┐
│  Shape: (400, 512, 512)    │       │  Shape: (400, 512, 512)    │
│                            │       │                            │
└────────────────────────────┘       └────────────────────────────┘
              │                                  │
              │      ┌──────────────┐            │
              └─────▶│   PSNR 计算   │◀───────────┘
                     └──────────────┘
                            │
                            ▼
                     PSNR = 38.5 dB
                     SSIM = 0.95
```

## 📊 代码关键调用链

```
train.py
    │
    ├── ▶ Dataset 构建
    │   │
    │   ├── train_list = os.listdir(path_img + 'train/1mm/')
    │   │                  └── ['CT00000000', 'CT00000001', ...]
    │   │
    │   └── train_set = train_Dataset(train_list)
    │       │
    │       └── __getitem__(idx)
    │           │
    │           └── in_model.get_train_img(path, case_name)
    │               │
    │               ├── sitk.ReadImage(f'train/1mm/{case}.nii.gz')  → HR
    │               ├── sitk.ReadImage(f'train/5mm/{case}.nii.gz')  → LR
    │               ├── random crop LR → (4, 256, 256)
    │               └── get corresponding HR → (16, 256, 256)
    │
    └── ▶ 训练循环
        │
        ├── for batch in train_loader:
        │   │
        │   ├── case_name, x, y = batch
        │   │   x: (B, 1, 4, 256, 256)    # LR input
        │   │   y: (B, 1, 16, 256, 256)   # HR label
        │   │
        │   ├── y_pred = model(x)
        │   │   └── (B, 1, 16, 256, 256)  # SR output
        │   │
        │   └── loss = L1Loss(y_pred, y)
        │
        └── ▶ 验证 (每 gap_val 个 epoch)
            │
            └── for batch in val_loader:
                │
                ├── case_name, crops, gt, pos = batch
                │   crops: (N, 4, 256, 256)   # N patches
                │   gt: (400, 512, 512)       # Full HR
                │   pos: [(z,y,x), ...]       # Patch positions
                │
                ├── for i, crop in enumerate(crops):
                │   pred_crop = model(crop)
                │   stitch_to_volume(pred_crop, pos[i])
                │
                ├── pred_volume: (400, 512, 512)
                │
                └── psnr = cal_psnr(pred_volume, gt)
```

## 🧮 尺寸变换总结

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          尺寸变换速查表                                       │
└─────────────────────────────────────────────────────────────────────────────┘

原始数据:
    5mm:  (Z, 512, 512)        ←── 输入 (低分辨率)
    1mm:  (5Z, 512, 512)       ←── 标签 (高分辨率)

训练时:
    随机裁剪后:
    5mm:  (4, 256, 256)        ──▶ 网络 ──▶  (16, 256, 256)  SR输出
    1mm:  (16, 256, 256)       ←── 标签对比 ──┘
    
    尺寸关系: 4 × 5 = 16 (Z方向5倍上采样)

验证时:
    原始: (80, 512, 512)       ──▶ 切块 ──▶  160 个 (4, 256, 256)
                                   │
                                   ▼
                              160 个 (16, 256, 256)
                                   │
                                   ▼
                              拼接 ──▶  (400, 512, 512)  SR输出
                              
    与 GT (400, 512, 512) 对比计算 PSNR
```
