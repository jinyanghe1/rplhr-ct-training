## 📋 开发规范 (新增)

### 最小化增量原则

⚠️ **强制要求**: 从此刻开始，所有修改必须遵守以下规范

| 规范 | 要求 | 违反后果 |
|------|------|----------|
| **单一修改** | 一次commit只修改一个模块/参数组 | 无法归因效果 |
| **变更限制** | 一次commit不超过**3处**修改 | 难以回滚 |
| **原子性** | 每个commit可独立回滚 | 依赖混乱 |

### 修改-验证-闭环

**强制流程**:
```
修改 → 训练5-10epoch → Benchmark对比 → 决策
                              ↓
                    提升>0.2dB → ✅ 保留
                    提升<0.2dB → ⚠️ 再验证/回滚
                    下降 → ❌ 立即回滚
```

**不回滚必须证明**（满足至少一条）:
1. PSNR提升 > 0.2 dB
2. 收敛速度提升 > 20%
3. Loss波动降低 > 30%
4. 视觉质量显著改善
5. 有文献支撑

**详细规范文档**: `DEVELOPMENT_SOP.md`
