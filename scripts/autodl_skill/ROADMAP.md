## 🎯 新策略原则

### 核心原则: 串行验证，快速收敛

```
┌─────────────────────────────────────────────────────────────┐
│  优化策略: 同类优化串行验证，有提升即跳过，无提升继续尝试       │
├─────────────────────────────────────────────────────────────┤
│  Phase 1: 超参数/学习策略优化 (当前阶段)                      │
│  ├── Step 1: Loss函数 → 验证 → 若提升≥0.5dB则进入Step 2       │
│  ├── Step 2: 数据增强 → 验证 → 若提升≥0.5dB则进入Step 3       │
│  ├── Step 3: 训练策略 → 验证 → 若提升≥0.5dB则进入Step 4       │
│  └── Step 4: 推理策略 → 验证 → 完成后进入Phase 2              │
│                                                             │
│  Phase 2: Backbone小模块 (Phase 1完成后)                     │
│  └── 添加即插即用模块 (Attention, RCAB, 特征融合等)           │
│                                                             │
│  Phase 3: Backbone架构升级 (若Phase 1+2后<25dB)              │
│  └── SwinIR 3D 适配                                         │
└─────────────────────────────────────────────────────────────┘
```

### 决策规则

| 验证结果 | 决策 |
|----------|------|
| 提升 ≥ 0.5 dB | ✅ **采纳**，跳过同类其他方案，进入下一阶段 |
| 提升 0.2-0.5 dB | ⚠️ **保留但不继续**，记录为备选，进入下一阶段 |
| 提升 < 0.2 dB | ❌ **放弃**，尝试同类下一个方案 |
| 下降 | ❌ **立即回滚**，尝试下一个方案 |

### 验证标准
- **快速验证**: 训练 20 epoch 观察趋势
- **完整验证**: 训练 50 epoch 确认最终效果
- **收敛判断**: Validation PSNR 3个epoch无提升则停止

### Step 1: Loss函数 + 训练稳定性优化 (当前)

**重要发现**: 当前基线应为 **20.11 dB** (Epoch 21)，而非20.01 dB！

**数据分析**:
```
Epoch  1: 17.67 dB  ← 初始
Epoch  6: 18.77 dB  ← 快速上升
Epoch 11: 19.86 dB  ← 持续上升
Epoch 16: 19.98 dB  ← 接近峰值
Epoch 21: 20.11 dB  ← 峰值! (实际基线)
Epoch 26: 19.93 dB  ← 开始下降/抖动
Epoch 71: 19.94 dB  ← 平台期震荡
```

**正确理解**: 
- ✅ 训练在Epoch 21达峰后自然进入平台期是正常的
- ✅ 继续训练到100 epoch不会带来提升
- 🎯 优化目标: 延长峰值前的上升期，减少峰值后的抖动

**验证顺序** (调整优先级):

| 优先级 | 优化项 | 预期增益 | 实现难度 | 状态 | 结果 |
|--------|--------|----------|----------|------|------|
| 1 | **EAGLELoss3D** | +0.5-1.0 dB | ⭐⭐ | ✅ | +0.10 dB |
| 2 | Charbonnier | +0.2-0.5 dB | ⭐ | ✅ | +0.10 dB |
| 3 | **Flip增强** | - | ⭐ | ✅ | +0.07 dB |

**结论**: 所有优化方法提升都 ~0.1 dB，当前架构可能已接近极限。

```
└── PSNR < 23 dB → 🔓 启动SwinIR适配
```
