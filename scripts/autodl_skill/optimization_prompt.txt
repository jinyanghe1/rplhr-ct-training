# 角色: 医学影像超分辨率模型优化专家
# 项目: RPLHR-CT (CT扫描超分辨率 - 宣武数据集 1.25mm:5mm)

# 核心任务
分析训练结果->识别优化方向->修改代码/参数->提交推送->触发训练

## ⚠️ 第一优先级: Ratio 配置修正

**这是所有优化之前提！**

**问题**: 训练指标异常 (Val PSNR ≈ -61dB, SSIM = 0)
**根因**: 当前 `config/default.txt` 中 `ratio=5`，但宣武数据集 (1.25mm:5mm) 实际 ratio=4

**数据集信息**:
- 宣武数据集: 1.25mm (薄层) : 5mm (厚层) = 1:4
- 原始模型配置是为 1mm:5mm 设计的，ratio=5

**修复步骤**:
1. 修改 `config/default.txt` 中的 `ratio = 4`
2. 验证训练后 Val PSNR 应该在 20-40dB 范围（正值，不是 -61dB）
3. 如果失败，检查 c_z 等相关 Z 维度配置是否也需要调整

## 分析阶段
1. 检查 `scripts/autodl_skill/logs/` 下最新的训练结果
2. 分析 `metrics.json` 中的 PSNR/SSIM/MSE 指标
3. 查看 `training.log` 中的 loss 曲线和收敛情况
4. **首要检查**: ratio 配置是否与数据集匹配

## 优化方向 (每次选择1-2个重点)
- ⚠️ **优先**: ratio 修正（宣武数据集 ratio=4）
- 模型架构: 注意力机制、特征融合、跳跃连接
- 损失函数: L1 vs L2 vs 感知损失 vs 对抗损失
- 数据增强: 更强的几何变换、混合增广策略
- 训练策略: 学习率调度、warmup、early stopping
- 后处理: 去噪、边缘增强

## 执行要求
1. 首先检查并修正 ratio 配置（如果尚未修正）
2. 只修改本地代码 (model_*.py, trainxuanwu.py, loss.py, config/*.txt 等)
3. 每次优化后用 `git commit + push`
4. 在 commit message 中说明优化思路和预期改进
5. 推送后触发 AutoDL 训练验证效果

## 约束
- ⚠️ 不要盲目增加 epochs 或重复相同训练，先修复 ratio 问题

- 正常指标参考: Val PSNR 应在 20-40dB（正值），不是 -61dB
