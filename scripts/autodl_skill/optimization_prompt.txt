## ⚠️ 最高优先级任务: 架构适配 ratio=4

**问题**: 当前模型配置 ratio=5，但宣武数据集 (1.25mm:5mm) 实际 ratio=4

**数据集信息**:
- 宣武数据集: 1.25mm (薄层) : 5mm (厚层) = 1:4
- split_manifest.json 中所有样本 ratio=4.0
- 当前 config/default.txt 中 ratio=5（错误）

**症状**: ratio=4 时报错
```
RuntimeError: shape '[1, 3, 4, 64, 4, 1]' is invalid for input of size 3328
```
这是因为 Z 维度变化导致与 Swin Transformer window_size 不匹配

**需要修改的内容**:
1. `config/default.txt`: ratio = 5 → 4
2. `data/dataset01_xuanwu/case_ratio_config.json`: 所有样本 ratio 5 → 4
3. **模型架构参数** (需要同时调整):
   - c_z (输入 Z 维度) 可能需要修改
   - Swin window_size 需要适配新的 Z 维度
   - 检查 swin_utils.py 中 BasicLayer/SwinTransformerBlock 的 window_size 参数

**验证方法**:
- 训练后 Val PSNR 应该在 20-40dB 范围（正值，不是 -61dB）
- 训练能正常启动，不报 RuntimeError

## 分析阶段

2. 分析 `metrics.json` 中的 PSNR/SSIM/MSE 指标

## 优化方向 (每次选择1-2个重点)
- ⚠️ **优先**: ratio=4 架构适配
- 模型架构: 注意力机制、特征融合、跳跃连接
- 损失函数: L1 vs L2 vs 感知损失 vs 对抗损失
- 数据增强: 更强的几何变换、混合增广策略
- 训练策略: 学习率调度、warmup、early stopping
- 后处理: 去噪、边缘增强
